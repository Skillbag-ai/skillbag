# SkillBag Sustainability

SkillBag is an open standard. The specification should remain freely readable,
implementable, and forkable.

Sustainability work should fund the shared goods around the standard, not put
the standard itself behind access control.

## Funding Principles

- The core specification remains open.
- Sponsorship does not buy private control over normative changes.
- Conformance should depend on tested behavior, not payment.
- Sponsors may receive public recognition and structured participation.
- Financial support should fund work that benefits the ecosystem.

## What Funding Supports

Funding may support:

- specification editing and release work
- a reference validator
- a conformance suite
- documentation and examples
- security review of installers and utility skills
- compatibility testing across agents
- community coordination and working group operations
- maintainer time for issue triage and review

## Support Paths

### Individual Donations

Individual donations are appropriate for users who want to support maintainer
time without needing invoices, governance participation, or formal benefits.

### Corporate Sponsorship

Corporate sponsorship is appropriate for companies that depend on portable
agent behavior and want the standard to mature.

Potential sponsor benefits:

- logo on the project site
- listing in `ADOPTERS.md`
- participation in public working group discussions
- early review of roadmap proposals
- public case study or implementation note

Sponsor benefits should be non-exclusive and should not grant veto power over
the standard.

### Working Group Membership

Working group membership is appropriate when several organizations want regular
technical coordination around interoperability, conformance, or future
standardization.

Membership should be separated from conformance. A member implementation still
needs to pass the same tests as any other implementation.

### Foundation or Consortium Hosting

If multi-vendor adoption becomes serious, SkillBag may move to a neutral host
or create a formal consortium structure.

This should be considered when the project needs:

- neutral IP and trademark stewardship
- formal corporate membership
- transparent standards process
- durable funding operations
- certification or conformance administration

## Commercial Services

Commercial services may be built around SkillBag, including:

- implementation consulting
- SkillBag workspace audits
- custom skill development
- hosted registries
- enterprise policy tooling
- conformance preparation

Commercial services should not imply exclusive ownership of the standard.

## Current Status

No formal sponsorship, membership, or certification program has launched yet.

Until a formal program exists, organizations can support the project by:

- adopting SkillBag publicly
- opening implementation feedback issues
- funding maintainer work directly
- sponsoring validator and conformance-suite development
- contributing utility skills and test fixtures
